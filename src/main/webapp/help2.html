Introduction

This system is intended to facilitate the registration of rental properties in Bloomington and issue permits for property owners. It provide the means to track the registration process, inspection, billing and other related activities. This system will provide the search engine and issue reports to measure performance. All authorized users will be able to view and navigate in the system, lookup permits, rental property owners, rental agents, complaint related to rental properties, .. etc.

This manual will give users an introduction to the Rental Processing System (RentPro). The system is written in Java Servlets. A web based user interface with Oracle Database as backend data store. It can be run on any computer with network connection and a web browser such as Netscape and MS Internet Explorer. The application objective was the ease of use and web enabled so that the HAND employee will be able to use it from his/her desktop without any specific installations on his/her computer.

This application came as a replacement for the old rental application that has been used in the City for a number of years. The two application can co-exist together even though the new one has more features built in, new data fields, new reports and an extensive search engine.

    Simple and Extensive Search for Permits

    The idea behind two different kinds of searche is due to the fact that the registration system is based on a number of tables in the database. The main table is the registration table. In addition there are the owner table, address table, zoning table, .. .etc. The simple search is devoted to search mainly in the registration table only where address or owner information are not required in the search process. While the extensive search is intended to be used for complicated queries when the address or owner info for examople are important part of the search. Since the extensive search is based on the projection of multiple tables, some records coould be repeated. For example a permit with two owners could show as two records one for each owner. Similary for two addresses. In simple search each record will show up only once.

    Which search to use? To answer this question consider the following;

  • if looking for a record with a known id then use the simple search.
  • if looking for records between two periods, use simple search too.
  • if looking permits associated with certain owner, agent, address, zone or any inputs that are not available in simple search then use the extensive search.
  • As a rule of thumb use simple search first, if you think this form will not give you the necessary fields to make your query then use the extensive search.
  • Simple search will work faster as the query is simpler.
  • You may get results in simple search that you may not get in extensive search.

    In the search forms most of the text fields are case insensitive. The option of full match (is), start with, ends with or contains is also provided to increase the possibilities of matching operations. Such as street name, owner name, owner address and notes fields all carry this feature.

    Caution Try not to use the search engine without any entries. If for any reason the "Browse" button without filling any field, this means the user is asking for all the records in the system. Since these count in a number of thousands, this may take long time to get the result or you may get partial results and the possibility of the browser being hang. In this case try to close your browser and restart again and try to avoid this situation in the futur. To overcome this problem, the user is adviced to enter a reasonble range of dates that cover no more than one year at a time such as 01/01/2000 to 01/01/2001 and so on.     Check Address

    A search engine for addresses. Has two main goals. To verify that certain address exists in the GIS master address. Second to check if certain address is already used in the rental system. If the rental system is used the corresponding permit ID's will be listed in the matching table where the user can jump to the corresponding permit.

    Search for street name is case insensitive and could be partial match whern used with the options of "start with", "ends with", or "contains" otherwise it is a complete match with "is" option.

    When searching for an address use as little information as possible, such as street name. To narrow your search you can fill other fields such as street number and direction. See Fig. 11 for more details.

    Fig. 11 Check Address Form

    Logout

    It is important for each user to Log Out from the system when he/she is done. This operation will release the resources used by the login session. In addition it will protect the data from being tempered with by curious users. The logout option is available on the left panel and also in most of the right frames in the bottom part of the page.

    Privileges

    Since this system will be used by different departments in the City at the same time, the need to manage this system is essential for its success. The following was considered in the design of this package;

  • All the departments except HAND are allowed to use the system to extract information that help the daily acitivities but are not allowed to do update or deletes. The authorized users in HAND department are the only authorized to do updates,inserts and deletes on this system according to the privileges granted to the users.
