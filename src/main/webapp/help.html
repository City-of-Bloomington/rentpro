Introduction

This system is intended to facilitate the registration of rental properties in Bloomington and issue permits for property owners. It provide the means to track the registration process, inspection, billing and other related activities. This system will provide the search engine and issue reports to measure performance. This system is desinged for the HAND department.

This manual will give users an introduction to the Rental Processing System (RentPro). The system is written in Java Servlets. A web based user interface with Oracle Database as backend data store. It can be run on any computer with network connection and a web browser such as Netscape and MS Internet Explorer. The application objective was the ease of use and web enabled so that the HAND employee will be able to use it from his/her desktop without any specific installations on his/her computer.

This application came as a replacement for the old rental application that has been used in the City for a number of years. The two application can co-exist together even though the new one has more features built, new data fields, new reports and an extensive search engine.

This course will walk you through this program by simple steps with snap shots of the real interface forms with the explanations of different options and what to expect.

    Add a New Permit or Registration

    Clicking on the first option in the left panel of options the user will be introduced to the 'New Registration" form as shown in Fig. 4.

    Handling Images

    From the registration form the user can click on the Inages button to open a floating window that will let the user upload a new image to this permit. A few points to consider before updoading a new image.

  • Download the image from the Digital Camera to your desktip.
  • Rename the image according to the following
    • do not include any space in the neme.
    • an initial name such as the inspector username or the road name of the property followed by two digits for month, followed by two digits for day, followed by two digits for sequence.
    • the image must have the extension as .jpg or .JPG. Other possible extension are .gif or .GIF but oftern these are not popular in digital cameras in the market but could be obtained by using softwares such Photoshop and others. Some sanners may give you this option too.
    • the image size should not exceed one meg of bytes.
    • The following are valid image names: roger110122.jpg, roger11.01.22.jpg, roger11_01_22.jpg, roger11.01.22.gif
    • The following are not valid image names: "roger" (no extension), "roger 11 01 22.jpg" (contain spaces), "roger110122.tif (unsupported format).
    • These are not good image names, roger.jpg (missing date and sequence), roger1101.jpg (missing sequence), roger11.1.1.jpg (need two digits for day and
  • Upload the image to form by clicking on the "Browse" button where you select the image you have renamed from your desktip
  • You can add any comments to this picture that will remind you of the purpose of this picture and what it presents and any other info that may be helpful.
  • Hit the "Save" button.
  • If any problem happens during the upload an error message will show up. Potential probems could be either the image name is too long or the image size is more than one meg of bytes. If the image was uploaded properly, it will show up in the form and will be added to the list of images of this property that will appear underneath this form. See Fig. 11.

    Simple and Extensive Search for Permits

    The idea behind two different kind of searches is due to the fact that the registration system is based on a number of tables in the database. The main table is the registration table. In addition there are the owner table, address table, zoning table, .. .etc. The simple search is devoted to search mainly in the registration table only where address or owner information are not required in the search process. While the extensive search is intended to be used for complicated queries when the address or owner info for examople are important part of the search. Since the extensive search is based on the projection of multiple tables, some records coould be repeated. For example a permit with two owners could show as two records one for each owner. Similary for two addresses. In simple search each record will show up only once.

    Which search to use? To answer this question consider the following;

  • if looking for a record with a known id then use the simple search.
  • if looking for records between two periods, use simple search too.
  • if looking permits associated with certain owner, agent, address, zone or any inputs that are not available in simple search then use the extensive search.
  • As a rule of thumb use simple search first, if you think this form will not give you the necessary fields to make your query then use the extensive search.
  • Simple search will work faster as the query is simpler.
  • You may get results in simple search that you may not get in extensive search.

    In the search forms most of the text fields are case insensitive. The option of full match, start with, ends with or contains is also provided to increase the possibilities of matching operations. Such as street name, owner name, owner address and notes fields all carry this feature.

    Caution Try not to use the search engine without any entries. If for any reason the "Browse" button without filling any field, this means the user is asking for all the records in the system. Since these count in a number of thousands, this may take long time to get the result or you may get partial results and the possibility of the browser being hang. In this case try to close your browser and restart again and try to avoid this situation in the futur. To overcome this problem, the user is adviced to enter a reasonble range of dates that cover no more than one year at a time such as 01/01/2000 to 01/01/2001 and so on.     Check Address

    A search engine for addresses. Has two main goals. To verify that certain address exists in the GIS master address. Second to check if certain address is already used in the rental system. If the rental system is used the corresponding permit ids will be listed in the matching table where the user can jump to the corresponding permit.

    Search for street name is case insensitive and could be partial match whern used with the options of "start with", "ends with", or "contains" otherwise it is a complete match with "is" option.

    When searching for an address use as little information as possible, such as street name. To narrow your search you can fill other fields such as street number and direction. See Fig. 16 for more details.

    Reports

    This form provides a list of options to select from. The options inclue (see Fig. 17 below);

  • List of addresses in the rental system.
  • List of invalid addresses.
  • Pulled Dates. List of permits with pulled dates and reasons arranged in date, pulled reason order order.
  • Pulled Dates with reasons. For certain reason the user will get all the permits within the date range of pull-date. If no reason is selected, the list will include all reasons for that period. The order is in reason, date order this time. The user can alwarys reorder the list according to the sort category needed.
  • List of agents in the system. Prefered to be combined with certain date range.
  • List of owners in the system. Prefered to be combined with certain date range.
  • New Cycle, actaully it means the list of new permits for this year.
  • Year inspections. List of inspections dates.
  • Year inspection types. List of inspections dates with their types.
  • Overdue bills. Show all the permits that have been issued a bill more than one month from now but have not been paid yet.
  • Properties managed by agent. List of all properties managed by certain agent (if the agent is selected from the list) otherwise the list will be all the properties managed by agents. Date fields can be ignored.
  • Properties managed by an owner. The owner id must be entered. Date fields should be ignored here as well.

    Dates fields should be used where it makes sense to be combined with. Such the list addressed could be combined with registered address. The option of permits with pull dates when checked the system will automatically mark the Pull date as the selected date type. Certain options, such as Overdue bills need not combined with any date since it does not make any sense to do so. Similary you do not to set dates for the options "Property managed by Owner" or Properties managed by agent".

    Logout

    It is important for each user to logout from the system when he/she is done. This operation will release the resources used by the login session. In addition it will protect the data from being tempered with by curious users. The logout option is available on the left panel and also in most of the right frames in the bottom part of the page.

    Privileges

    Since this system will be used by different departments in the City at the same time, the need to manage this system is essential for its success. The following was considered in the design of this package;

  • All the departments except HAND are allowed to use the system to extract information that help the daily acitivities but are not allowed to do update or deletes. The authorized users in HAND department are the only authorized to do updates,inserts and deletes on this system according to the privileges granted to the users.
